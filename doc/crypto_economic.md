# DeFi 生态核心概念

## 一、AMM 与 DEX 流动性池

### 1.1 池子的构成

Uniswap V2 的 BTC/USDT 池，核心是 **AMM（自动化做市商）**。没有订单簿，没有买卖挂单，只有两份代币储备：

```
池子里有：10 BTC  +  500,000 USDT
```

价格由一个简单的公式决定：

```
x * y = k
x = BTC 数量, y = USDT 数量, k = 常数
```

你拿 1 BTC 换 USDT，池子算出新的状态：

```
(10 + 1) * (500,000 - USDT_OUT) = 5,000,000
→ USDT_OUT = 454,545
```

得到 454,545 USDT ≈ 45,454 USDT/BTC。而市场价是 50,000，**滑点 = (50,000 - 45,454) / 50,000 ≈ 9%**。

### 1.2 滑点完全由池子深度决定

| 池子总价值 | 卖 1 BTC 的滑点 |
|-----------|---------------|
| $5,000 | ~50% |
| $50,000 | ~5% |
| $50,000,000 | ~0.1% |

几千 U 的池子没有实际交易价值——滑点大到没法做市。

### 1.3 Uniswap V3 的区别

V3 允许 LPs 把流动性集中在一个价格区间内（如 49,000 ~ 51,000），用更少的资金提供更深的深度。但需要主动管理，价格跌出区间就白干了。

### 1.4 DEX 池子的来源

**流动性提供者（LPs）创建的。** 任何人可以往任意交易对池子里注入代币，成为 LP，赚取交易手续费分成（每笔 0.01% ~ 1%，通常 0.3%）。

为什么有人往里放钱：

| 动机 | 说明 |
|------|------|
| 赚手续费 | 主流币大池交易量大，收入稳定 |
| 做市策略 | V3 集中流动性，比 CEX 做市更省资本 |
| 项目方自建 | 新代币上线自己放流动性吸引交易 |
| 收益农耕 | 协议额外发治理代币奖励 LPs |

为什么有些池子只有几千 U：**手续费覆盖不了无常损失的风险。** 没人用的池子自然没人放钱。

### 1.5 一个代币可以有无数个 DEX 池子

**因为池子的创建不需要任何人批准。** 谁都能在 Uniswap 上用任意两个代币创建一个池。所以同一个代币在不同链、不同 DEX、甚至同一个 DEX 上都有不同的池子，各自独立运行。

DEX 的池子是一个**开放市场**，谁都能开。发行成本极低——一段 ERC-20 合约加一笔 Gas 费就能诞生一个代币，再花一笔 Gas 就能建池。这是 DEX 上流动性分散且大量池子深度极浅的根源。

## 二、CEX vs DEX 的流动性差异

### 2.1 DEX 流动性被 CEX 虹吸

| 维度 | DEX | Binance CEX |
|------|-----|-------------|
| 典型流动性 | 几千 U | 订单簿深度数百万 U |
| 用户群体 | 散户、小单 | 做市商、机构、大单 |
| 价格发现 | 跟随 CEX | **主导定价** |

做市商和机构资金在 CEX 提供深度，DEX 只是"小渠"。一个 $100M 市值的代币可能在 Uniswap 上只有一个 $5K 的池子——因为没有人需要用 DEX 来交易它。

### 2.2 链上暴涨会反馈到交易所吗？

**不一定，取决于 DEX 池子的大小：**

- DEX 池子 $5K → 花 $10K 就能把价格推高 100x
- 套利者最多只能从这 $5K 池子里获利 → 利润空间极小
- 池子太小，套利不足以拉动币安的价格
- **真正的价格发现发生在流动性所在的地方**——即币安的订单簿

### 2.3 DEX 的核心价值不在交易量

DEX 流动性的真正价值不是被散户用来买卖的，而是被**协议**使用的：

- Aave 用 Uniswap 池做清算
- 机枪池用 DEX 做策略调仓
- 做市商用 DEX 做对冲
- 新币首发（LBP、Fair Launch）必须用 DEX

这些不需要大量流动性，只需要**可用**。

## 三、公链与智能合约

### 3.1 图灵完备

图灵完备 = 能算任何可计算的问题。条件分支（`if`）+ 循环（`for`/`while`）。

**比特币故意不做成图灵完备**——因为图灵完备的代价是**停机问题**。如果合约里有死循环怎么办？以太坊的解法是 **Gas**：

```
每执行一条指令扣一次 Gas
Gas 用完 → 强制终止
钱照扣，但不保证结果
```

Gas 计量把"这个程序会不会死循环"从编译时转移到运行时——你不需要证明你的合约没有死循环，你只需要付钱让它跑。

### 3.2 图灵完备的链可以做什么

| 功能 | 需要 | 比特币 | 以太坊 |
|------|------|--------|--------|
| 转账 | 简单脚本 | ✅ | ✅ |
| 多重签名 | 条件分支 | ✅ | ✅ |
| AMM 自动做市 | 循环 + 复杂状态 | ❌ | ✅ |
| 借贷协议 | 状态机 + 回调 | ❌ | ✅ |

**清算为什么需要图灵完备**——因为需要**有记忆的状态**：

比特币是 UTXO 模型，每笔交易独立执行，没有全局变量。以太坊是**账户模型**，合约可以持久化存储状态变量（如抵押品价值、借款金额、清算阈值），任何人都可以调用 `liquidate()` 函数检查条件并执行清算。

### 3.3 所有图灵完备的公链都有 AMM

| 链 | AMM 协议 |
|-----|---------|
| Ethereum | Uniswap, Curve, Balancer |
| BSC | PancakeSwap |
| Solana | Raydium, Orca |
| Arbitrum | Uniswap, Camelot |
| Base | Uniswap, Aerodrome |
| Polygon | QuickSwap |
| Avalanche | Trader Joe |

### 3.4 没有 CEX，区块链还有价值吗？

**有价值，但价格会暴跌。**

比特币的价值不在 CEX 或 DEX 上，在它作为一个**无需许可的结算网络**——任何人只要运行一个节点就能验证账本，任何国家都关不掉。CEX 消失，价格会暴跌，但比特币不会归零。它会退化为一个"慢速"的价值存储网络：OTC 做市商会填坑，链上原子交换会进化。

DeFi 的真正价值也不是交易量，而是**可组合的去中心化金融基础设施**：

- 一个 USDC 在 Compound 上作为抵押品赚利息，同时又作为 Uniswap 的流动性凭证，再被用作结算资产——**同一笔钱同时在三个协议里干活**
- CEX 做不到这个，因为没有哪个交易所会让你的保证金同时去别的协议打工

## 四、做市商经济模型

### 4.1 做市商在 DeFi 怎么赚钱

**LP 手续费收入**是核心来源。每笔交易收 0.3%，按 LP 份额比例分。但真正成本是**无常损失**。

### 4.2 无常损失

假设你在 ETH/USDC 池里放了 10 ETH + 500,000 USDC（ETH = $50K，总投入 $1M）。

ETH 涨到 $60K → 套利者来买走便宜的 ETH，直到池内价格跟上。你取出时：

```
总价值 ≈ $990K  ← 什么都没做，亏了 $10K
```

### 4.3 做不做这个币的决策

```
预期收益 = 手续费收入 - 无常损失预期 - Gas 成本
如果 > 0 → 做；如果 < 0 → 不做
```

| 因子 | 对决策的影响 |
|------|-------------|
| 交易量 | 越大越好，手续费收入高 |
| 波动率 | 越小越好，无常损失低 |
| 池子现有深度 | 越浅越容易被大单吃掉利润 |
| 代币相关性 | base/quote 越相关（如稳定币对），无常损失越低 |
| Gas 费 | 小池子赚的还不够付 Gas |

5,000 U 的 BTC/USDT 池子没人做：BTC 波动率太大，无常损失预期远高于那点手续费收入。**只有稳定币对或深度足够的池子，做市才划算。**

## 五、代币价值

### 5.1 如何扩展代币的价值

代币本身没有价值，价值来自于**别人为什么要持有它**：

| 方向 | 示例 |
|------|------|
| 治理权 | 持有可投票决定协议参数（UNI） |
| 经济权益 | 分润、折扣、质押收益（CAKE） |
| 稀缺性机制 | 销毁、回购、通缩模型 |
| 刚性需求 | 用代币才能使用某个服务 |
| 激励相容 | 让别人有钱可赚才会进来 |

**价值不是从代码里长出来的，是从使用场景里长出来的。** 代码可以复制，场景不能。

### 5.2 量化识别有前景的代币

在已有的 ClickHouse 数据基础设施上，可以从这些信号维度量化分析：

| 维度 | 数据来源 | 信号 |
|------|---------|------|
| 解锁 vs 价格 | `token_unlock` + 价格历史 | 解锁后价格是否坚挺 → 真实买盘 |
| DEX 流动性 / CEX 交易量 | `token_liquidity` + 币安数据 | 比值越低 → 做市商主导 |
| 资金费率 + 持仓量 | `funding_rate` + `open_interest` | 解锁前后费率突变 → 聪明钱布局 |
| 持仓集中度 | `token_holders` | 解锁后前 10 地址是否减少 |

**好的量化信号只能帮你排除坏标的，不能帮你找到百倍币。** 百倍币靠的是叙事 + 时机 + 运气，数据能做的只是不让你在解锁前一天冲进去接盘。
